> 본 문서는 팀 프로젝트 **ERP Program**에서 제가 담당한 역할과 기여 내용을 정리한 문서입니다.

## 역할 및 기여
- **역할**: 프론트엔드
- **기여 내용**
  - ERP 프로그램의 기본적인 화면 레이아웃 설계 및 구현
  - 사용자 클릭 및 입력에 따른 동적 UI 처리
  - Fetch API를 활용한 서버와의 데이터 송수신 구현
  - 기능 별 리팩토링을 통해 가독성 및 유지보수성 개선

- **언어**: JavaScript
- **스타일링**: CSS3

## 트러블 슈팅 및 배운 점

### 1. `clamp()`를 활용한 반응형 웹 처리
- **문제**
  - vw, % 등의 동적 단위를 사용했지만 화면 해상도에 따라 일부 UI 요소가 어색하게 보이는 문제가 발생
- **해결**
  - CSS `clamp()`를 적용하여 화면 크기에 따라 자연스럽게 변하도록 수정
- **결과**
  - 다양한 해상도 환경에서도 비교적 안정적인 화면을 유지할 수 있었음

### 2. jQuery에서 바닐라 JavaScript로 전환
- **상황**
  - 외부 라이브러리 의존도를 줄이기 위해 기존 jQuery 코드를 바닐라 JavaScript로 변경
- **적용**
  - `querySelector`, `querySelectorAll`, `forEach` 활용
  - jQuery의 HTML 코드를 `insertAdjacentHTML`, `innerHTML` 등으로 대체
- **배운 점**
  - `querySelectorAll`의 반환값인 NodeList를 통해 forEach문과 같이 사용하여 요소가 없는 경우에도 에러가 발생하지 않도록 처리

### 3. 조건에 따른 UI 스타일링
- **구현**
  - 특정 조건에 따라 `.main_data` 클래스를 가진 요소에 `.active` 클래스 추가 및 제거
- **결과**
  - 사용자 동작에 따라 화면이 즉각적으로 반응하는 UI를 구현할 수 있었음

### 4. 리팩토링
- **문제**
  - UI 처리, 이벤트 로직, 데이터 통신 코드가 한 파일에 있어 가독성이 떨어짐
- **해결**
  - 기능별로 파일을 분리하고 중복 코드를 함수로 정리
- **배운 점**
  - JS 파일을 분리하면 모듈 간 참조 관계에 따라 일부 값이 초기화되지 않아 `undefined`가 발생할 수 있음

### 5. 클로저 활용
- **상황**
  - 클릭된 데이터 정보를 유지한 채 footer 영역 내 상세 내용에 해당 데이터를 표시해야 하는 요구사항 발생
- **적용**
  - 클로저를 활용해 각 항목의 데이터를 유지
- **결과**
  - 데이터 클릭 시 footer 영역 내 상세내용에 해당 정보가 정상적으로 출력
 
### 6. Fetch API 비동기 통신
- **상황**
  - Spring 백엔드 서버와 실시간으로 데이터를 주고받아 DB 내용을 반영해야 함
- **적용**
  - Fetch API로 서버와 통신
  - 응답 데이터를 JSON 배열로 변환 후, 각 프로퍼티에 접근하여 데이터 사용
    
    | HTTP 메서드 | 역할 |
    | :--- | :--- |
    | **POST** | 데이터 저장 |
    | **GET** | 데이터 조회 |
    | **PUT** | 데이터 수정 |

## 느낀 점
- 프론트엔드와 백엔드 간 데이터 명칭과 타입을 초기 단계에서 맞추는 것이 전체 기능 안정성에 매우 중요하다는 점을 느꼈음
- 전공 수업에서 배운 내용을 실제 프로젝트에 적용하고 부족한 부분은 자료 조사와 실습을 통해 보완함
- 이 경험을 바탕으로 현재 진행 중인 **TimeCapsule 프로젝트**에서는 초기 설계 단계부터 구조를 더 구체적으로 잡아가고 있음
